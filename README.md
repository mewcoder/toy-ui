> 代码提交规范
- feat	新增一个功能
- fix	修复一个 Bug
- docs	文档变更（更新文档、Demo）
- ci	发布控件版本
- style	代码风格变更（不影响功能，解决代码冲突、eslint 校验修改等）
- refactor	重构（即不是新增功能，也不是修改 bug 的代码变动）
- chore	开发工具变动（框架改动、eslint 规则变动等）
- test	新增测试
- perf	改善性能
- revert	代码回退（如果当前 commit 用于撤销以前的 commit）
